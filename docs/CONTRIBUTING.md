## Как начать

### 1. Форк репозитория
```bash
# Нажмите кнопку "Fork" на GitHub
# Клонируйте ваш форк
git clone https://github.com/YOUR_USERNAME/EchoBrowser.git
cd EchoBrowser
```

### 2. Создайте ветку для вашей функции
```bash
git checkout -b feature/your-feature-name
# или для исправления ошибок
git checkout -b fix/bug-description
```

### 3. Делайте коммиты с понятными сообщениями
```bash
git commit -m "Описание изменения"
```

### 4. Пушьте изменения
```bash
git push origin feature/your-feature-name
```

### 5. Создайте Pull Request
- Перейдите на GitHub в ваш форк
- Нажмите "New Pull Request"
- Опишите ваши изменения
- Дождитесь проверки

## Требования к коду

### Стиль кода
```python
# Используйте PEP 8
# Хорошо:
def function_name(parameter1, parameter2):
    """Описание функции"""
    result = parameter1 + parameter2
    return result

# Плохо:
def functionName(p1,p2):
    return p1+p2
```

### Документирование
```python
class MyClass:
    """Описание класса"""
    
    def method(self, param1: str) -> int:
        """
        Описание метода
        
        Args:
            param1: Описание параметра
        
        Returns:
            int: Описание возвращаемого значения
        """
        pass
```

    def setUp(self):

    def test_functionality(self):
        result = self.feature.do_something()
        self.assertEqual(result, expected_value)

## Процесс проверки кода

1. **Проверка стиля**: Код должен следовать PEP 8
2. **Проверка тестов**: Все тесты должны проходить
3. **Проверка документации**: Новый код должен быть документирован
4. **Проверка архитектуры**: Новые модули должны соответствовать структуре проекта
5. **Одобрение**: Минимум одного владельца проекта

## Сообщение об ошибках

### Как сообщить об ошибке:

1. **Проверьте**, не была ли ошибка уже зарегистрирована
2. **Создайте Issue** с названием вида: `[BUG] Краткое описание ошибки`
3. **Опишите**:
   - Ваша ОС и версия Python
   - Версия браузера
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Скриншоты (если применимо)

### Пример:
```markdown
## Описание ошибки
При нажатии на кнопку "Закрыть" браузер падает.

## Шаги для воспроизведения
1. Открыть браузер
2. Открыть новую вкладку
3. Нажать кнопку X на вкладке

## Ожидаемое поведение
Вкладка должна закрыться

## Фактическое поведение
Приложение аварийно завершает работу

## Среда
- ОС: Windows 10
- Python: 3.13
- Версия: 0.1.0
```

## Запрос на функцию

1. **Создайте Issue** с названием вида: `[FEATURE] Названи функции`
2. **Опишите**:
   - Что должна делать функция
   - Почему она необходима
   - Как она улучшает браузер
   - Примеры использования

- Менеджер закладок в меню Инструменты

## Конвенция коммитов

Используйте понятные сообщения коммитов:

```
<type>: <description>

<body>

<footer>
```

### Типы коммитов:
- `feat`: Новая функция
- `fix`: Исправление ошибки
- `docs`: Изменения в документации
- `style`: Стилистические изменения (форматирование, точки с запятой и т.д.)
- `refactor`: Переструктурирование кода без изменения функциональности
- `perf`: Улучшение производительности
- `test`: Добавление или изменение тестов
- `chore`: Изменения конфигурации, зависимостей и т.д.

### Примеры:
```
feat: добавить поддержку закладок

Реализована полная система управления закладками:
- Добавление/удаление закладок
- Сохранение в JSON файл
- UI для управления

Fixes #123
```

```
fix: исправить краш при закрытии последней вкладки

При попытке закрыть последнюю открытую вкладку браузер крашился.
Теперь вместо закрытия создается новая пустая вкладка.

Fixes #456
```

## Процесс разработки

### 1. Выбор задачи
- Посмотрите на open issues
- Выберите задачу, которая вас интересует
- Напишите комментарий, что вы начали над ней работать

### 2. Разработка
- Создайте новую ветку
- Реализуйте функцию/исправление
- Пишите тесты во время разработки
- Часто коммитьте

### 3. Тестирование
```bash
# Запустить все тесты
python -m pytest tests/

# Запустить конкретный тест
python -m pytest tests/test_my_feature.py

# Запустить с выводом покрытия
python -m pytest --cov=src tests/
```

### 4. Проверка кода
```bash
# Проверить стиль (PEP 8)
pip install flake8
flake8 src/

# Проверить типы (если используются аннотации)
pip install mypy
mypy src/
```

### 5. Подготовка PR
- Убедитесь, что все тесты проходят
- Обновите документацию
- Напишите понятное описание PR
- Проверьте, что нет конфликтов с основной веткой
